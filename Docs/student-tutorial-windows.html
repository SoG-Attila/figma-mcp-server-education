Step 3: Get Figma Access Token (~5 min)

We need a special password (token) to access Figma programmatically.

Create Your Token

  1. Go to figma.com and log in
  2. Click your profile picture (top-right) ‚Üí Settings
  3. Scroll down to "Personal access tokens" section
  4. Click "Create a new personal access token"
  5. Give it a name: MCP Server
  6. Check these permissions:
    • ‚úÖ File content: Read
    • ‚úÖ File metadata: Read
  7. Click "Generate token"
  8. COPY THE TOKEN immediately - you'll only see it once!
  9. Paste it in Notepad/TextEdit and keep it open
‚ö†Ô∏è Never share this token publicly! It's like a password to your Figma account.

You Should Have Two Things Now

  1. Figma Token: Starts with figd_ (from above)
  2. File ID: From Step 0 (the alphanumeric code from your Figma URL)

Keep both saved in a text file - you'll need them in the next step!

Step 4: Create Your Server Files (~20 min)

We'll create two files:

  1. .env - Stores your secrets (token & file ID)
  2. server.py - The code that connects to Figma

Create .env File

üçé macOS

In PowerShell (make sure you're still in FigmaMCPServer folder and (venv) is showing):

notepad .env

Click "Yes" to create the file. Then paste this, replacing with YOUR token and file ID:

FIGMA_TOKEN=your_token_here FIGMA_FILE_ID=your_file_id_here

Example with real values:

FIGMA_TOKEN=figd_abc123xyz789_your_actual_token_here FIGMA_FILE_ID=ABC123XYZ456789

Save (Ctrl+S) and close Notepad.

Create server.py File

Download the server code from your instructor OR the GitHub repository.

Save the server.py file directly in your FigmaMCPServer folder.

File location should be:

  • Windows: C:\Users\YourName\Documents\FigmaMCPServer\server.py
  • Mac: /Users/YourName/Documents/FigmaMCPServer/server.py

Test the Server

Back in PowerShell/Terminal (make sure (venv) is showing):

üçé macOS
python server.py
‚úÖ If the cursor just blinks (no errors), it works! Press Ctrl+C to stop it.

If you see errors:

  • "FIGMA_TOKEN not found" - Check your .env file has the correct format
  • "API Error 403" - Your token is invalid, create a new one
  • "ModuleNotFoundError" - Make sure venv is activated and libraries installed

Configure Claude Desktop

We need to tell Claude Desktop where to find your MCP server.

üçé macOS

1. Find your Python path:

In PowerShell (with venv activated):

where.exe python

Copy the FIRST line shown (the one with "venv" in it). Example:

C:\Users\YourName\Documents\FigmaMCPServer\venv\Scripts\python.exe

2. Edit config file:

notepad %APPDATA%\Claude\claude_desktop_config.json

3. Paste this template (replace the paths with YOUR paths):

{ "mcpServers": { "figma-text-extractor": { "command": "C:\\Users\\YourName\\Documents\\FigmaMCPServer\\venv\\Scripts\\python.exe", "args": ["C:\\Users\\YourName\\Documents\\FigmaMCPServer\\server.py"] } } }
‚ö†Ô∏è Critical for Windows:
  • Use double backslashes \\ NOT single \
  • Both paths must point to YOUR actual files
  • The first path ends with python.exe
  • The second path ends with server.py

Save (Ctrl+S) and close Notepad.

üÜò Common Problems & Solutions

Problem: "Could not connect to MCP server"

Solutions:

  • ‚úÖ Check your paths in config file - no typos!
  • ‚úÖ Windows: Use double backslashes \\ not single \
  • ‚úÖ Make sure (venv) was active when you installed libraries
  • ‚úÖ Restart Claude Desktop COMPLETELY (Quit, not just close)
  • ‚úÖ Try opening Claude Desktop from scratch

Problem: "Figma API Error: 403"

Your token is invalid or expired.

Solution:

  1. Go to Figma and create a new token
  2. Update your .env file with the new token
  3. Restart Claude Desktop

Problem: "ModuleNotFoundError: No module named 'mcp'"

Libraries weren't installed in the virtual environment.

Solution:

# Make sure you're in the project folder cd ~/Documents/FigmaMCPServer # Mac cd ~\Documents\FigmaMCPServer # Windows # Activate venv source venv/bin/activate # Mac .\venv\Scripts\Activate # Windows # Reinstall pip install mcp requests python-dotenv

Problem: Virtual environment won't activate (Windows)

Execution policy is blocking scripts.

Solution:

  1. Open PowerShell as Administrator (right-click ‚Üí Run as administrator)
  2. Run: Set-ExecutionPolicy RemoteSigned
  3. Type Y and press Enter
  4. Close admin PowerShell and try activating venv again in regular PowerShell
